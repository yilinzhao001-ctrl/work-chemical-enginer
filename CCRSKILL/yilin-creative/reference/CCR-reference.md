## 工艺基础

### 连续催化重整 (CCR) 概述

连续催化重整是催化重整的一种类型，特点是催化剂在反应器间连续移动。主要工艺：
- **UOP 重叠式**：国内应用较多
- **IFP/Axens 并列式**：投资、占地大，下有料斗

### 反应类型

| 反应 | 描述 |
|------|------|
| 六元环烷烃脱氢 | 大多数能转成芳烃 |
| 五元环烷烃异构 | 5到6比较慢，主要由5~6决定 |
| 烷烃脱氢环化 | 先变5元环，戊二烯是积碳中间体 |
| 烷烃异构化 | 提高重整生成油的辛烷值 |
| 加氢裂化/氢解 | 高温高压下促进，重整低压抑制 |

### 主要操作条件

```
压力: 0.35~1.5 MPa
温度: 480~530°C
氢烃摩尔比: 2~8
体积空速: 1~3 h^-1
```

### 原料要求

**原料来源**：
- 常减压直馏石脑油（大部分）
- 加氢裂化石脑油（芳潜高，一般不进预加氢）
- 焦化石脑油（S、N含量高，耗氢量高）
- 催化裂化石脑油（烯烃多，含S、N杂质）

**杂质限制**：
- 硫化物：< 0.5 μg/g（损害催化剂活性）
- 氮化物：需预加氢严格控制
- 氯化物：电脱盐电流会超
- 砷：永久失活

**原料控制指标**：
- N+2A（环烷烃+芳烃体积分数）
- 芳烃潜含量 (Ar%)
- 终馏点：基本控制175°C，加30~40°C为汽油终馏点205°C

### 产品分布

- 氢气（副产品，纯度利于PSA提纯）
- 燃料气/液化气
- C5+ 抽提（苯、二甲苯、抽余油）
- 戊烷（异戊烷、正戊烷）

## 平台操作

### 登录信息

### 建模流程

1. **上传模型文件**：
   - 有时提示"没有找到脚本文件"，使用 `direct from` 方式上传可解决
   - 检查模型文件路径是否正确

2. **mdb 文件处理**：
   - 首次建模需生成 mdb 文件
   - 选择 YANSHAN 单位制
   - 后续版本如未生成 mdb，输出单位会与 Input 不一致
   - 如遇取数问题，删除 mdb 文件可能解决

3. **调整单位**：
   - 检查 Inputs/Outputs 的单位集是否统一为 YANSHAN

### 采样任务

- **单次采样数量**: 5000 samples
- **采样时间**: 约 18 小时
- **典型问题**: 部分台机会失败，需分批处理

### 常见问题处理

| 问题 | 解决方案 |
|------|----------|
| 提示没有业务文件 | 使用 direct from 方式上传 |
| 收结果报错 input-out | 数据库缺少记录，找IT改代码 |
| 部分样本不收敛 | 分批提交，每批5-7台 |
| 模型打不开 | 重启虚拟机 |
| 输出数据单位不对 | 生成 mdb 文件并选择 YANSHAN 单位制 |
| 一致性错误 | 点计算多次会收敛，或调整进料波动方式 |

## 模型变量说明

### 变量列说明

| 列名 | 含义 |
|------|------|
| Var | 变量名 |
| description | 变量描述 |
| Type | 类型（如 modification） |
| Relation | 关系（如 Less than） |

### 关键变量示例

| 变量名 | 描述 | 公式 |
|--------|------|------|
| SO_C16201SC_Reb_Q | 脱戊烷塔釜再沸器负荷 | `SO_C16201SC_MinRefluxRatio*1.2` |
| SO_C16401SC_Reb_Q | 脱丁烷塔釜再沸器负荷 | `SI_C16401_ExternalRefluxRatio` |

### 配方因子

- **回流比倍数**: 1.2 ~ 1.5 倍最小回流比
- 成功率较低时可调整公式

### 产品切割

- **C5+切割**: 产品分割点，通常在 C4/C5 之间
- **PONA**: 烷烃、烯烃、环烷烃、芳烃分析

## 问题诊断

### 收敛问题

1. **脱戊烷塔不收敛**
   - 调整输入回流比为计算得到的最小回流比的 1.2 倍
   - 或调整顶温、冷凝温度、回流率范围

2. **脱丁烷塔不收敛**
   - 检查灵敏板温度分布
   - 调整冷凝温度数据分布
   - 控冷凝温度后收敛率可提升约 10%

3. **脱乙烷塔不收敛**
   - 检查模型中 Inputs 表塔的压力链接是否正确
   - 注意塔的顺序，后面的塔收敛建立在前面塔基础上

### 单位问题

- 输出的流量单位为 kg/s，压力单位为 kPa，温度单位不变
- 如输出与 Input 不一致，生成 mdb 文件时选择 YANSHAN 单位制

### 平台卡顿

- 关闭后重新打开模型文件
- 修改配置文件后需重新上传
- 注意保存工作进度
